# 古典ビット回路と量子ビット回路の比較ページ

GitHub Pages でそのまま公開するなら、React や JSX ではなく、静的な HTML / CSS / JavaScript に分けるのが安全です。

この repo では、比較ページを次の 3 ファイルで実装しています。

- [classical-quantum.html](classical-quantum.html)
- [classical-quantum.css](classical-quantum.css)
- [classical-quantum.js](classical-quantum.js)

以下は、JSX 版メモを GitHub Pages 向けの plain Markdown / HTML / JavaScript の説明へ書き直したものです。

## ねらい

- 左に古典回路、右に量子回路を並べる
- ボタンでゲートを追加し、回路の流れを可視化する
- 古典側は 0 か 1 の確定値を返す
- 量子側は状態、確率、測定履歴を表示する
- ビルド不要で、そのまま github.io に置ける形にする

### JavaScript

JavaScript では、ゲート定義、状態、計算関数、描画関数、イベント登録を分けます。

```js
const classicalGates = [
  {
    name: "NOT",
    desc: "0 を 1 に、1 を 0 に反転",
    kind: "single",
    apply(bit) {
      return 1 - bit;
    }
  },
  {
    name: "BUF",
    desc: "そのまま通す",
    kind: "single",
    apply(bit) {
      return bit;
    }
  }
];

const appState = {
  classicalInput: 0,
  classicalCircuit: ["NOT"],
  quantumInput: "0",
  quantumCircuit: ["H"],
  measurements: []
};

function computeClassical() {
  let bit = appState.classicalInput;
  const steps = [{ label: "入力", value: bit }];

  appState.classicalCircuit.forEach((gateName) => {
    const gate = classicalGates.find((item) => item.name === gateName);
    bit = gate.apply(bit);
    steps.push({ label: gateName, value: bit });
  });

  return { bit, steps };
}

function renderClassical() {
  const result = computeClassical();
  document.querySelector("#classicalResultValue").textContent = String(result.bit);
}

function measureQuantum() {
  const probabilities = computeQuantum();
  const randomValue = Math.random() * 100;
  const measured = randomValue < probabilities.p0 ? "0" : "1";
  appState.measurements.unshift(measured);
  appState.measurements = appState.measurements.slice(0, 20);
  renderQuantum();
}

function renderAll() {
  renderClassical();
  renderQuantum();
}

document.querySelector("#measureQuantum").addEventListener("click", measureQuantum);
renderAll();
```

## JSX 版からの置き換え方

React / JSX でやっていたことは、GitHub Pages 用の素のページでは次のように置き換えます。

- 状態管理は、フレームワークの hook ではなく `appState` のような普通のオブジェクトへ置き換える
- 再計算は、メモ化 hook ではなく `computeClassical()` や `computeQuantum()` のような関数へ置き換える
- UI 部品は、ライブラリのコンポーネントではなく標準の `button` 要素へ置き換える
- 属性名はフレームワーク専用のものではなく、通常の HTML 属性へ置き換える
- クリック処理は DOM の `addEventListener("click", ...)` で登録する
- アニメーション用の props は CSS transition か通常の DOM 更新へ置き換える

## この repo での最終形

この比較ページは、すでに GitHub Pages 向けの plain HTML / CSS / JavaScript に移してあります。

- [classical-quantum.html](classical-quantum.html) がページ構造
- [classical-quantum.css](classical-quantum.css) が見た目
- [classical-quantum.js](classical-quantum.js) が回路計算、測定、ダイアグラム描画

そのため、公開時に React のビルドや JSX 変換は不要です。普通の静的ファイルとしてそのまま置けます。
